## Project Overview

**Site name:** भारतीय ज्ञानप्रणाली व वारसा / Elements of Indian Knowledge Systems  
**GitHub repo:** https://github.com/vivek-sovani/IKS  
**Live URL:** https://vivek-sovani.github.io/IKS  
**Source book:** `Elements of IKS - Compiled version Kindle ready  .pdf` (258 pages)  
**Author of translation:** Vivek Sovani (Marathi translation from English original)

**Purpose:** A bilingual (Marathi + English) educational website presenting the book's content as accessible web articles, one article per chapter. Each article has a summarised version (Phase I, current) and an advanced reading version (Phase II, future).

**Languages:** Every page is fully bilingual. All content elements have both `data-lang="mr"` (Marathi) and `data-lang="en"` (English) versions. Language preference is saved in `localStorage` as `iks-lang`.

## Two-Tier Authoring Workflow

### Articles 01–04: Hand-coded HTML
These articles were written directly as HTML files. Use `articles/01/index.html` as the reference template when you need to understand the HTML structure, but do **not** hand-code new articles.

### Articles 05+: content.json → build.js (preferred)
Starting with article 05, all new articles use a data-driven build pipeline:

1. Author the article as `articles/NN/content.json`
2. Run `node build.js NN` to generate `articles/NN/index.html`
3. Run `node build.js all` to rebuild every article that has a `content.json`

**Always use content.json for any new articles (07 onwards).**

## Architecture Decisions

### Self-contained vs. shared files

- **On GitHub Pages (live site):** Each article uses external CSS (`/IKS/assets/css/`) and loads sidebar via `fetch('/IKS/shared/sidebar.html')`. This works because GitHub Pages serves over HTTP.

- **Best practice:** Run a local server during development:
  ```bash
  cd ~/Projects/IKS
  npx serve . -p 3000
  ```
  Then open `http://localhost:3000` — external CSS and fetch() both work.

### Language toggle

All bilingual elements use `data-lang="mr"` or `data-lang="en"` attributes:
```html
<p data-lang="mr">मराठी मजकूर येथे.</p>
<p data-lang="en" style="display:none;">English text here.</p>
```

The `IKS.setLang()` function in `nav.js` toggles `display:none` on all these elements. Language is saved to `localStorage('iks-lang')`.

### nav.js — IKS global API

`nav.js` exports the global `IKS` object with these methods:

| Method | Description |
|---|---|
| `IKS.setLang('mr'|'en')` | Switch display language, persist to localStorage |
| `IKS.getLang()` | Return current language string |
| `IKS.toggleSidebar()` | Open/close sidebar |
| `IKS.closeSidebar()` | Close sidebar |
| `IKS.toggleSection(id)` | Expand/collapse a sidebar section (e.g. `'c0'`) |
| `IKS.openSection(id)` | Open a sidebar section |
| `IKS.setActiveArticle(num)` | Highlight active article link and open its section |
| `IKS.init()` | Attach event listeners; called on DOMContentLoaded and after sidebar fetch |

Each article page calls `IKS.init()` twice: once automatically on `DOMContentLoaded`, and again after the sidebar HTML is injected via fetch.

## content.json Schema

This is the authoritative format for all new articles. See `articles/05/content.json` or `articles/06/content.json` as complete examples.

### Top-level fields

```json
{
  "id": "NN",
  "section": "c0|c1|c2|c3",
  "titleMr": "मराठी शीर्षक",
  "titleEn": "English Title",
  "readingTime": "५",
  "readingTimeEn": "5",
  "metaDesc": "Optional meta description (160 chars). Falls back to summary.mr.",
  "summary": {
    "mr": "2–3 sentence summary in Marathi.",
    "en": "2–3 sentence summary in English."
  },
  "body": [ /* array of block objects — see below */ ],
  "nav": {
    "prev": { "id": "NN", "titleMr": "...", "titleEn": "..." },
    "next": { "id": "NN", "titleMr": "...", "titleEn": "..." }
  }
}
```

Omit `nav.prev` for the first article; omit `nav.next` for the last.

### Block types

#### `section` — main content section

```json
{
  "type": "section",
  "headMr": "मराठी शीर्षक",
  "headEn": "English Heading",
  "paragraphs": [
    { "mr": "मराठी परिच्छेद.", "en": "English paragraph." }
  ],
  "insight": {
    "mr": "Key philosophical insight in Marathi (optional).",
    "en": "Key philosophical insight in English (optional)."
  }
}
```

One `insight` per article maximum. Put it in the section where it fits best.

#### `termBox` — Sanskrit term definition

```json
{
  "type": "termBox",
  "word": "संस्कृत शब्द",
  "roman": "IAST Transliteration",
  "defMr": "मराठी व्याख्या. मूळ शब्दाचा अर्थ सांगा.",
  "defEn": "English definition. Explain etymology."
}
```

Use 1–2 termBoxes per article for the most important Sanskrit terms.

#### `concepts` — key concepts grid

```json
{
  "type": "concepts",
  "items": [
    {
      "term": "संस्कृत शब्द",
      "roman": "Transliteration",
      "defMr": "मराठी व्याख्या",
      "defEn": "English definition"
    }
  ]
}
```

Use 2–4 items. Renders as a responsive card grid.

#### `curiosity` — curiosity box

```json
{
  "type": "curiosity",
  "mr": "रोचक तथ्य किंवा प्रश्न — मराठी.",
  "en": "Interesting fact or question — English."
}
```

#### `activity` — reflection exercise

```json
{
  "type": "activity",
  "mr": ["पहिली कृती.", "दुसरी कृती.", "तिसरी कृती."],
  "en": ["First activity.", "Second activity.", "Third activity."]
}
```

#### `phase2` — advanced reading teaser (always last in body)

```json
{
  "type": "phase2",
  "mr": "मराठी — सखोल अध्ययनाचे वर्णन.",
  "en": "English — description of what advanced reading will cover."
}
```

#### `infographic` — SVG diagram

```json
{
  "type": "infographic",
  "svgType": "flow|wheel|bars",
  "titleMr": "मराठी शीर्षक",
  "titleEn": "English Title",
  "captionMr": "मराठी कॅप्शन.",
  "captionEn": "English caption.",
  "... svgType-specific fields ..."
}
```

See SVG diagram schemas below.

## SVG Diagram Types

All diagrams are bilingual: `data-lang="mr"` and `data-lang="en"` `<g>` layers are generated automatically from the JSON data.

### `flow` — left-to-right step diagram

Use for processes, timelines, sequences.

```json
{
  "svgType": "flow",
  "titleMr": "...", "titleEn": "...",
  "captionMr": "...", "captionEn": "...",
  "steps": [
    {
      "mr": ["लाइन १", "लाइन २"],
      "en": ["Line 1", "Line 2"],
      "roman": "Optional IAST",
      "subMr": ["उप-मजकूर"],
      "subEn": ["Sub-text"],
      "highlight": true
    }
  ],
  "examples": {
    "headMr": "उदाहरणे:", "headEn": "Examples:",
    "mr": ["उदाहरण १", "उदाहरण २"],
    "en": ["Example 1", "Example 2"]
  }
}
```

`highlight: true` on one step makes it the visually prominent center step. `examples` is optional — adds a bottom row.

### `wheel` — central concept with four quadrant panels

Use for a central idea radiating outward into four dimensions/categories.

```json
{
  "svgType": "wheel",
  "titleMr": "...", "titleEn": "...",
  "captionMr": "...", "captionEn": "...",
  "center": {
    "mr": "केंद्र",
    "en": "Center",
    "roman": "Sanskrit romanisation",
    "subMr": "उप-मजकूर",
    "subEn": "Sub-text"
  },
  "quadrants": [
    {
      "position": "top-left|top-right|bottom-left|bottom-right",
      "color": "green|gold|purple|saffron",
      "headMr": "...", "headEn": "...",
      "subMr": "...", "subEn": "...",
      "itemsMr": ["आयटम १", "आयटम २", "आयटम ३"],
      "itemsEn": ["Item 1", "Item 2", "Item 3"]
    }
  ]
}
```

Provide exactly 4 quadrants; each at a distinct `position`.

### `bars` — state-comparison bar chart

Use for spectrums, progression of states, comparative scales.

```json
{
  "svgType": "bars",
  "titleMr": "...", "titleEn": "...",
  "captionMr": "...", "captionEn": "...",
  "barLabelMr": "बार लेबल", "barLabelEn": "Bar label",
  "showInfinity": true,
  "states": [
    {
      "mr": "अवस्था नाव",
      "en": "State name",
      "roman": "Optional",
      "subMr": "उप-मजकूर", "subEn": "Sub-text",
      "sub2Mr": "दुसरा उप-मजकूर", "sub2En": "Second sub",
      "highlight": true,
      "barWidth": 120
    }
  ],
  "callout": {
    "mr": ["शीर्षक:", "ओळ १", "ओळ २"],
    "en": ["Heading:", "Line 1", "Line 2"],
    "mrRight": ["उजवा स्तंभ शीर्षक:", "ओळ"],
    "enRight": ["Right column heading:", "Line"]
  }
}
```

Provide 3–4 states. `highlight: true` on the key/climax state. `callout` is optional.

## Infographic Guidelines

Each article should have **1–2 SVG infographics** embedded as `<figure class="art-image">`.

### SVG rules
- `viewBox="0 0 700 320"` (always 700 wide; increase height if needed)
- Background: `fill="#FAF6EE"` (ivory, matching page)
- Fonts: `font-family="'Noto Sans Devanagari',sans-serif"` for Marathi text
- Fonts: `font-family="'Crimson Pro',Georgia,serif"` for English/Sanskrit italic
- Always include `role="img"` and `<title>` for accessibility
- Keep labels concise — SVG does not wrap text
- Bilingual: two `<g>` layers, `data-lang="mr"` and `data-lang="en" style="display:none"`

### Infographic type selection

| Article type | SVG type |
|---|---|
| Single concept with four dimensions | `wheel` |
| Process, sequence, timeline | `flow` |
| Spectrum, states, comparison | `bars` |

### Image prompts file
For articles with `img/` folder, create `articles/NN/img/image-prompts.txt` with 4 prompts:
1. Infographic prompt (for NotebookLM or Canva)
2. Philosophical illustration (for Midjourney/DALL-E)
3. Presentation slide (for Google Slides/Canva)
4. WhatsApp share card (1080×1080px)

## Marathi Style Guide

### Grammar principles
- Use **शुद्ध मराठी** — avoid unnecessary English words
- Sentence structure: Subject + Object + Verb (SOV) — standard Marathi
- Use proper विभक्ती (case suffixes): ला, ने, चा/ची/चे, मध्ये, साठी, वर
- Use **होते/असते/आहे** correctly — do not mix tenses within a paragraph
- Prefer active voice over passive
- Use **म्हणजे** for definitions, not **म्हणतात**

### Vocabulary preferences

| Avoid | Prefer |
|-------|--------|
| experience (English) | अनुभव |
| process | प्रक्रिया |
| concept | संकल्पना |
| basically | मूलतः / थोडक्यात |
| very important | अत्यंत महत्त्वाचे |
| etc. | इत्यादी |
| mechanism | यंत्रणा |
| connection | संबंध / नाते |

### Sanskrit terms in Marathi text
- Always write the Sanskrit term first: **आनंद (Ananda)**
- Follow with meaning in parentheses on first use: **धर्म (ऋतम् राखण्याची प्रक्रिया)**
- Use italic for Sanskrit shlokas/verses: *शृण्वन्तु विश्वे...*
- Do NOT translate Sanskrit technical terms — explain them

### Sentence flow
- Keep sentences under 30 words
- One idea per sentence
- Use connecting words: **म्हणून, परंतु, शिवाय, याशिवाय, त्याचप्रमाणे, तरीही**
- 2–4 sentences per paragraph
- End sections with a synthesizing statement

### Common mistakes to avoid
- Do NOT write: "हे basically सांगते की..." (mixed language)
- Do NOT write: "ते म्हणतात की..." without naming the source
- Do NOT write overly long compound sentences with multiple subordinate clauses
- Do NOT use the same sentence opener twice in a row

## Content Guidelines for Articles

### What each article should contain
1. **Opening paragraph** — relatable real-life hook (everyday example)
2. **Core concept** — what does this term mean in IKS framework
3. **Mechanism or process** — how does it work
4. **Key insight** — the central philosophical point (goes in insight block)
5. **Practical significance** — why this matters to a modern reader
6. **Curiosity** — one surprising fact or comparison
7. **Activity** — 2–3 reflective exercises

### Content source
- **Primary source:** The PDF book `Elements of IKS - Compiled version Kindle ready  .pdf`
- Read the relevant chapter(s) carefully before writing
- **Summarised version:** Compress the book content to essential points — do not just translate verbatim
- **Do not add** content not present in the book
- **Do not omit** key concepts from the book's chapter

### Article length
- Summary version (Phase I): 600–900 words per language
- 4–6 content sections
- 1–2 Sanskrit term boxes
- 1 infographic (wheel, flow, or bars)
- 1 curiosity box
- 1 activity box

## How to Add a New Article

### Step 1 — Read the book chapter
```bash
# Extract relevant pages from PDF (adjust page numbers)
pdftotext -f PAGE_START -l PAGE_END "Elements of IKS - Compiled version Kindle ready  .pdf" -
```

### Step 2 — Create folder and content.json
```bash
mkdir -p articles/NN
# Create articles/NN/content.json using the schema above
# Use articles/05/content.json or articles/06/content.json as full examples
```

### Step 3 — Build the HTML
```bash
node build.js NN
# Verify: articles/NN/index.html is created
```

### Step 4 — Create image prompts (optional)
```bash
mkdir -p articles/NN/img
# Create articles/NN/img/image-prompts.txt with 4 prompts
```

### Step 5 — Update sidebar.html
Add the new article link in the correct `<div class="sb-art-list">` in `shared/sidebar.html`:
```html
<a class="sb-art-link done" href="/IKS/articles/NN/" data-art="NN">
  <span class="sb-art-num">NN</span>
  <span class="sb-art-titles">
    <span class="mr-t">मराठी शीर्षक</span>
    <span class="en-t">English Title</span>
  </span>
</a>
```
Use `class="sb-art-link done"` for published (shows ✓). Use `class="sb-art-link"` for draft.

### Step 6 — Update site.json
Change `"status": "draft"` → `"status": "published"` for article NN.

### Step 7 — Commit and push
```bash
git add articles/NN/ shared/sidebar.html site.json
git commit -m "Add Article NN: [English Title]"
git push -u origin main
```
